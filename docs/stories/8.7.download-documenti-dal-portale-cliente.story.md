# Story 8.7: Download Documenti dal Portale Cliente

**As a** Cliente, **I want** to download my quote and invoice PDFs from the portal, **so that** I can archive my repair documentation.

## Acceptance Criteria

### AC-1
_Derived from epic-details AC-1._
**Given** fattura `id=8` belongs to the authenticated portal customer `clienteId=5` and a valid portal Bearer token is provided
**When** the customer calls `GET /api/portal/documenti/fattura/8/pdf`
**Then** the API returns HTTP `200` with header `Content-Type` containing `application/pdf` and `Content-Disposition` attachment filename matching the fattura document name

### AC-2
_Derived from epic-details AC-2._
**Given** preventivo `id=5` belongs to the authenticated portal customer `clienteId=5` and a valid portal Bearer token is provided
**When** the customer calls `GET /api/portal/documenti/preventivo/5/pdf`
**Then** the API returns HTTP `200` with header `Content-Type` containing `application/pdf` and `Content-Disposition` attachment filename matching the preventivo document name

### AC-3
_Derived from epic-details AC-3._
**Given** requested fattura `id=8` belongs to another customer and the caller is authenticated as portal customer `clienteId=5`
**When** the caller requests `GET /api/portal/documenti/fattura/8/pdf`
**Then** the API returns HTTP `403` with `error.code="FORBIDDEN"` and no PDF bytes in response body

### AC-4
_Derived from epic-details AC-4._
**Given** no portal Bearer token is provided for a preventivo download request
**When** the caller requests `GET /api/portal/documenti/preventivo/5/pdf`
**Then** the API returns HTTP `401` with error message `Unauthorized`

## Task Breakdown

- [ ] Task 1 (AC-1, AC-2, AC-3, AC-4): add portal endpoints `GET /api/portal/documenti/fattura/:id/pdf` and `GET /api/portal/documenti/preventivo/:id/pdf` in `packages/backend/src/routes/auth.ts` using the same bearer-token guard and portal error mapping used by existing `/api/portal/*` routes.
- [ ] Task 2 (AC-1, AC-2, AC-3): add a portal document download use case in `packages/backend/src/services/auth-service.ts` that resolves customer identity from access token and enforces ownership checks before exposing PDF content.
- [ ] Task 3 (AC-1): expose/reuse fattura PDF retrieval in `packages/backend/src/services/fatture-service.ts` for portal-safe download payload (`fileName`, `content`) without bypassing domain validation.
- [ ] Task 4 (AC-2): expose/reuse preventivo PDF retrieval in `packages/backend/src/services/preventivi-service.ts` for portal-safe download payload (`fileName`, `content`) without bypassing ownership constraints.
- [ ] Task 5 (AC-1, AC-2, AC-3, AC-4): implement ATDD coverage in `packages/backend/src/__tests__/portal-documenti-download.atdd.spec.ts` for success (fattura/preventivo), forbidden cross-customer access, and unauthenticated request behavior.
